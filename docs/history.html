<h1 id="browser-plugin">Browser plugin</h1> <p>The browser plugin will automatically update your browser URL and state on route changes. It will also listen to popstate events (triggered by back and forward buttons and manual URL changes).</p> <p>This plugin uses HTML5 history API and therefore is not compatible with browsers which don&#39;t support it. Refer to <a href="http://caniuse.com/#search=history">caniuse.com</a> for browser compatibility.</p> <p>It adds a bunch of functions to work with full URLs: <code>router.buildUrl(routeName, routeParams)</code> and <code>router.matchUrl(url)</code>. It also decorates the start function so you don&#39;t have to supply any start path (it extracts it from the current URL).</p> <pre><code class="lang-javascript javascript">import browserPlugin from &#39;router5/plugins/browser&#39;; const router = createRouter() .usePlugin(browserPlugin({ useHash: true })) .start(); </code></pre> <p>For more information on how to install, look at <a href="/docs/installation.html">Get started</a>.</p> <h3 id="options">Options</h3> <ul> <li><code>forceDeactivate</code>: default to <code>true</code>, meaning <code>canDeactivate</code> handlers won&#39;t get called on popstate events. It is not recommended to set it to <code>false</code>.</li> <li><code>useHash</code></li> <li><code>hashPrefix</code></li> <li><code>base</code>: the base of your application (the part to add / preserve between your domain and your route paths).</li> <li><code>preserveHash</code>: whether to preserve the initial hash value on page load (default to <code>true</code>, only if <code>useHash</code> is <code>false</code>)</li> <li><code>mergeState</code>: whether to keep any value added in history state by a 3rd party or not (default to <code>false</code>)</li> </ul>
