<h1 id="navigation">Navigation</h1> <blockquote> <p>After configuring your routes, you need to <em>enable navigation</em> by starting your router instance.</p> </blockquote> <h2 id="starting-your-router">Starting your router</h2> <pre><code class="lang-javascript javascript">const myRouter = createRouter([ { name: &#39;home&#39;, path: &#39;/home&#39; }, { name: &#39;about&#39;, path: &#39;/about&#39; }, { name: &#39;contact&#39;, path: &#39;/contact&#39; } ]); myRouter.start(&#39;/home&#39;); </code></pre> <blockquote> <p>When using <code>.start()</code>, you should supply a starting path or state except if you use the browser plugin (the current URL will automatically be used).</p> </blockquote> <p>Invoking the <code>.start(startPathOrState[, done])</code> function will:</p> <ul> <li>Attempt to navigate to <code>startPathOrState</code></li> <li>Attempt to match the current URL if no <code>startPathOrState</code> was provided, or navigation failed</li> <li>Attempt to navigate to the default route if it could not match the provided start path or if <code>startPathOrState</code> was not provided / failed</li> </ul> <p>And will:</p> <ul> <li>Enable navigation</li> </ul> <p>Providing a starting state is designed to be used for universal JavaScript applications: see <a href="/docs/universal-applications.html">universal applications</a>.</p> <h2 id="defining-a-default-route">Defining a default route</h2> <p>A default route can be set in <code>createRouter</code> options. The following example will cause your application to navigate to <code>/about</code>:</p> <pre><code class="lang-javascript javascript">var myRouter = createRouter([ { name: &#39;home&#39;, path: &#39;/home&#39; }, { name: &#39;section&#39;, path: &#39;/:section&#39; } ], { defaultRoute: &#39;section&#39; defaultParams: {section: &#39;about&#39;} }) .start(function (err, state) { /* ... */ }); </code></pre> <p>A callback can be passed to start and will be invoked once the router has transitioned to the default route.</p> <h2 id="navigating-to-a-specific-route">Navigating to a specific route</h2> <p>Router5 exposes the following method: <code>navigate(routeName, routeParams, opts)</code>. This method has to be called for navigating to a different route: <strong>clicks on links won&#39;t be intercepted by the router</strong>.</p> <pre><code class="lang-javascript javascript">myRouter.navigate(&#39;section&#39;, {section: &#39;contact&#39;}); // Will navigate to &#39;/contact&#39; </code></pre> <p><strong>Forcing a reload</strong></p> <p>When trying to navigate to the current route nothing will happen unless <code>reload</code> is set to <code>true</code>.</p> <pre><code class="lang-javascript javascript">myRouter.navigate(&#39;section&#39;, {section: &#39;contact&#39;}, {reload: true}); </code></pre> <p><strong>Replacing current state</strong></p> <p>Set <code>replace</code> to true for replacing the current state in history when navigating to a new route. Default behaviour is to add an entry in history.</p> <pre><code class="lang-javascript javascript">myRouter.navigate(&#39;section&#39;, {section: &#39;contact&#39;}, {replace: true}); </code></pre> <h2 id="knowing-when-a-transition-has-been-successful-or-not">Knowing when a transition has been successful or not</h2> <p>Like for <code>.start()</code>, <code>.navigate()</code> accepts a callback (last argument):</p> <pre><code class="lang-javascript javascript">myRouter.navigate(&#39;route&#39;, function (err, state) { /* ... */ }) </code></pre> <h2 id="stopping-your-router">Stopping your router</h2> <p>At any time you can stop (pause) a router and it will prevent any navigation. To resume, simply invoke <code>.start()</code> again.</p> <pre><code class="lang-javascript javascript">myRouter.stop(); </code></pre>
