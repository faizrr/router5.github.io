<h1 id="configuring-routes">Configuring routes</h1> <blockquote> <p>There are a few ways to add routes to your router. You can specify your routes when creating a router instance and / or use chainable <code>add</code> and <code>addNode</code> methods to add routes.</p> </blockquote> <h2 id="defining-your-routes-as-pojos">Defining your routes as POJOs</h2> <p>You can define your routes as a flat array or nested array of routes. When using a flat array of routes, nested route names need to have their full name specified.</p> <p>Route objects also accept the following properties:</p> <ul> <li><code>canActivate</code>: a method to control whether or not the route node can be activated (see <a href="/docs/preventing-navigation.html">Preventing navigation</a>)</li> <li><code>forwardTo</code>: if specified, the router will transition to the forwarded route instead. It is useful for defaulting to a child route, or having multiple paths pointing to the same route.</li> </ul> <p><strong>Flat array of routes</strong></p> <pre><code class="lang-javascript javascript">const routes = [ { name: &#39;users&#39;, path: &#39;/users&#39;}, { name: &#39;users.view&#39;, path: &#39;/list&#39;}, { name: &#39;users.list&#39;, path: &#39;/view&#39;} ]; </code></pre> <p><strong>Nested arrays of routes</strong></p> <pre><code class="lang-javascript javascript">const routes = [ { name: &#39;users&#39;, path: &#39;/users&#39;, children: [ { name: &#39;view&#39;, path: &#39;/list&#39;}, { name: &#39;list&#39;, path: &#39;/view&#39;} ]} ]; </code></pre> <h2 id="adding-routes-to-your-router">Adding routes to your router</h2> <p>You can add all your routes at once using <code>createRouter</code> or <code>router.add</code>.</p> <p><strong>createRouter(routes, options)</strong></p> <pre><code class="lang-javascript javascript">const router = createRouter(routes, options); </code></pre> <p><strong>add(routes)</strong></p> <p><code>.add()</code> accepts single nodes, flat or nested arrays.</p> <pre><code class="lang-javascript javascript">myRouter.add({ name: &#39;about&#39;, path: &#39;/about&#39; }); // Or myRouter.add([ {name: &#39;about&#39;, path: &#39;/about&#39;}, {name: &#39;contact&#39;, path: &#39;/contact&#39;}, ]); </code></pre> <p><strong>addNode(name, path[, canActivate])</strong></p> <p>You can add routes node by node, specifying a node name and its segment path.</p> <pre><code class="lang-javascript javascript">var router = createRouter() .addNode(&#39;users&#39;, &#39;/users&#39;) .addNode(&#39;users.view&#39;, &#39;/view/:id&#39;) .addNode(&#39;users.list&#39;, &#39;/list&#39;); </code></pre> <h2 id="configuring-the-root-node-path">Configuring the root node path</h2> <p>At the top of your tree of routes, there is an unamed node called the root node. Its path is empty and can be configured using <code>router.setRootPath(path)</code>. It can be used for example to list a number of allowed query parameters for all routes in strict query parameters mode (ter.setRootPath(&#39;?param1&amp;param2&#39;)`).</p>
