<h1 id="dependencies-injectables-">Dependencies (injectables)</h1> <blockquote> <p>When using lifecycle methods (<code>canActivate</code>, <code>canDeactivate</code>), middleware or plugins, you might need to access specific objects from your application: a store, a specific API, etc... You can pass their reference to router5 and they will be passed alongside your router instance.</p> </blockquote> <p>You can register all dependencies at once, or one by one.</p> <pre><code class="lang-js">const router = createRouter(routes, options, dependencies); </code></pre> <pre><code class="lang-js">router.setDependencies({ store, api }); // or router.setDependency(&#39;store&#39;, store); router.setDependency(&#39;api&#39;, api); </code></pre> <p>You can retrieve your current dependencies references using <code>getDependencies()</code>.</p> <p>Lifecycle methods (<code>canActivate</code>, <code>canDeactivate</code>), middleware or plugins will be called with them:</p> <pre><code class="lang-js">const plugin = (router, dependencies) =&gt; ({ /* onStart() {}, onStop() {}, onTransitionStart() {}, ... */ }); </code></pre> <pre><code class="lang-js">const canActivate = (router, dependencies) =&gt; (toState, fromState, done) { /* ... */ } </code></pre> <pre><code class="lang-js">const middleware = (router, dependencies) =&gt; (toState, fromState, done) { /* ... */ } </code></pre>
