<h1 id="path-syntax">Path syntax</h1> <blockquote> <p><em>router5</em> uses <a href="https://github.com/troch/path-parser">path-parser</a> for parsing, matching and generating URLs</p> </blockquote> <h2 id="defining-parameters">Defining parameters</h2> <p>Four parameter types are supported:</p> <ul> <li><code>:param</code>: url parameters</li> <li><code>;matrix</code>: matrix parameters</li> <li><code>*splat</code>: for parameters spanning over multiple segments. Splat parameters are greedy and could swallow a large part of your URL. It is recommended to handle with care and to ONLY use on routes without children.</li> <li><code>?param1&amp;param2</code> or <code>?:param1&amp;:param2</code>: query parameters</li> </ul> <h2 id="constrained-parameters">Constrained parameters</h2> <p>Url and matrix parameters can be constrained with a regular expression. Backslashes need to be escaped.</p> <ul> <li><code>:param&lt;\\d+&gt;</code> will match numbers only for parameter param</li> <li><code>;id&lt;[a-fA-F0-9]{8}&gt;</code> will match 8 characters hexadecimal strings for parameter id</li> </ul> <p>Constraints are also applied when building paths: when passing incorrect params to <code>.navigate()</code>, an error will be thrown.</p> <h2 id="absolute-nested-paths">Absolute nested paths</h2> <p>You can define absolute nested paths (not concatenated with their parent&#39;s paths). Note that absolute paths are not allowed if any parent of a node has parameters.</p> <pre><code class="lang-js">const router = createRouter([ { name: &#39;admin&#39;, path: &#39;/admin&#39; }, { name: &#39;admin.users&#39;, path: &#39;~/users&#39; } ]); router.buildPath(&#39;admin.users&#39;); // &#39;/users&#39; </code></pre>
