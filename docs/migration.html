<h1 id="migrating-from-0-x-to-1-x">Migrating from 0.x to 1.x</h1> <blockquote> <p><strong>router5@1.0.0</strong> now uses plugins for managing browser history and handling listeners. Installling and using <code>router5-listeners</code> and <code>router5-history</code> will result in an almost identical behaviour.</p> </blockquote> <h2 id="new-features">New features</h2> <ul> <li>New <code>autoCleanUp</code> option for automatically deregistering components when their associated route is no longer active (also applies to route node listeners)</li> <li>New <code>canDeactivate</code> method as a shortcut for <code>registerComponent</code></li> <li>Plugins and built-in logger plugin</li> <li>Support for multiple middleware functions</li> </ul> <h2 id="breaking-change">Breaking change</h2> <ul> <li>Transition errors are now objects with a code property, and not strings</li> <li><code>onTransition()</code> is now deprecated in favour of <code>useMiddleware()</code></li> <li>history is now managed by a plugin (<code>router5-history</code>)</li> <li>listeners is now managed by a plugin (<code>router5-history</code>), API remains the same</li> </ul> <h2 id="code-example">Code example</h2> <pre><code class="lang-javascript javascript">import { Router5 } from &#39;router5&#39;; import historyPlugin from &#39;router5-history&#39;; import listenersPlugin from &#39;router5-listeners&#39;; const router = new Router5() .addNode(&#39;home&#39;, &#39;/home&#39;) .usePlugin(historyPlugin()) .usePlugin(listenersPlugin()) // Development helper .usePlugin(Router5.loggerPlugin()) .start(); </code></pre> <pre><code class="lang-javascript javascript">router.canDeactivate(&#39;home&#39;, true); // is the equivalent of router.registerComponent(&#39;home&#39;, { canDeactivate: function () { return true; } }); </code></pre>
