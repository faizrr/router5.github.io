<h1 id="universal-isomorphic-apps">Universal/Isomorphic Apps</h1> <blockquote> <p>JavaScript applications which run both client-side and server-side, typically sharing all or most of the same codebase.</p> </blockquote> <p><em>Router5</em> is just as capable on the server versus in the browser. This enables you to reuse the same routes for both your client-side navigating and your server-side pre-render.</p> <p>This is essentially done via two steps:</p> <ol> <li><strong>Server-side</strong> - Pass to your router the current URL (since there&#39;s no <code>location</code> object on the server).</li> <li><strong>Client-side</strong> - Pass to your router a starting state (via the <code>done</code> callback of <em>Step 1</em>) so that your router doesn&#39;t try to transition to the already activated page.</li> </ol> <h4 id="below-is-a-more-in-depth-tutorial">Below is a more in-depth tutorial</h4> <h2 id="create-your-router-client-server-">Create your router (client &amp; server)</h2> <p>You can use the same code for configuring your router on both client and server sides. The history plugin, for example, can be safely used on Node.js and in browsers.</p> <pre><code class="lang-js">const createRouter = require( &#39;router5&#39; ).default; const listenersPlugin = require( &#39;router5/plugins/listeners&#39; ); const browserPlugin = require( &#39;router5/plugins/browser&#39; ); function createRouter() { return createRouter([ { name: &#39;home&#39;, path: &#39;/home&#39; }, { name: &#39;about&#39;, path: &#39;/about&#39; }, { name: &#39;contact&#39;, path: &#39;/contact&#39; }, { name: &#39;404&#39;, path: &#39;/404&#39; } ], { trailingSlash: true, defaultRoute: &#39;404&#39; }) .usePlugin(browserPlugin({ useHash: false })) .usePlugin(listenersPlugin()) } export default createRouter </code></pre> <h2 id="server-side-routing">Server-side Routing</h2> <blockquote> <p>This example is an <a href="http://expressjs.com/">Express</a> with <a href="http://paularmstrong.github.io/swig/">Swig</a> application. Make changes were needed to suit your preferred frameworks.</p> </blockquote> <p>For universal applications, you need to:</p> <ul> <li>Create a new router instance for each request, using the request URL</li> <li>Send the state to the client and start your router with this initial state</li> </ul> <p><strong><code>server.js</code></strong></p> <pre><code class="lang-javascript javascript">import express from &#39;express&#39;; import createRouter from &#39;router5&#39;; import swig from &#39;swig&#39;; const app = express(); // Swig is used for templating in this example // Use what you are comfortable with app.engine( &#39;html&#39;, swig.renderFile ); app.set( &#39;view engine&#39;, &#39;html&#39; ); app.set( &#39;views&#39;, &#39;./views&#39; ); app.get( &#39;*&#39;, ( req, res ) =&gt; { // Create a new router for each request const router = createRouter(); router.start( req.originalUrl, function done( error, state ) { if ( error ) { res.status( 500 ).send( error ); } else { res.send(/* Use your router state, send some HTML! */ ); } }); } ); app.listen( 8080, function logServerStart() { console.log( &#39;Server is listening on port 8080...&#39; ); } ); </code></pre> <p><strong><code>base.html</code></strong></p> <pre><code class="lang-html">&lt;!doctype html&gt; &lt;html lang=&quot;en-US&quot;&gt; &lt;head&gt; &lt;title&gt;Example Server-side Routing&lt;/title&gt; &lt;/head&gt; &lt;body&gt; &lt;script src=&quot;/js/router.js&quot;&gt;&lt;/script&gt; &lt;script type=&quot;text/javascript&quot;&gt; /** * Load the App&#39;s inital state from the server * @type {JSON} */ var initialState = JSON.parse(&#39;{{ initialState | safe }}&#39;); /** * Start our Router * @param {Error} error Router start error * @param {Object} state State Object * @return {undefined} */ app.router.start(initialState, function(error, state) { if (error) console.error(&#39;router error&#39;, error); }); &lt;/script&gt; &lt;/body&gt; &lt;/html&gt; </code></pre> <p>From here forth, you can continue to use router5 as if it was a regular Single-Page Application.</p> <h3 id="performance">Performance</h3> <p>A new router should be created on each request. There is no evidence of performance issues, however router5 includes a cloning mechanism to reduce the time it takes to create a router. Create a base router, and clone it for each request.</p> <pre><code class="lang-js">const baseRouter = createRouter(/* ... */); const router = baseRouter.clone(dependencies); </code></pre> <h3 id="important-to-remember-pass-in-an-object">Important to Remember - Pass in an Object</h3> <p>It is important to remember that <code>router.start()</code> <strong>does NOT</strong> parse your starting state for you. If you pass in a <code>String</code> instead of an <code>Object</code> your router will attempt to navigate to the path of that string.</p>
